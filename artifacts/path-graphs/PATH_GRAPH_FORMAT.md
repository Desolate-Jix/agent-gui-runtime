# Path Graph Format — nav_path_graph_v1

Navigation path graph JSON schema. Each file records the interface navigation trace of one target application (`app_name`), generated by the agent-gui-runtime browser panel.

## File naming convention

```
<app_name>_path_graph.json
```

Examples:
- `MouseTesterWeb_path_graph.json`
- `Notepad_path_graph.json`

## Top-level structure

```jsonc
{
  "format_version": "nav_path_graph_v1",   // schema version
  "app_name": "MouseTesterWeb",            // target application name
  "saved_at": "2026-06-02T15:30:00.000Z",  // ISO 8601 save timestamp
  "summary": "3 pages, 2 transitions",     // human-readable summary
  "pages": [ /* ... */ ],                  // visited interface pages
  "transitions": [ /* ... */ ]             // click actions between pages
}
```

## `pages[]` — Interface pages

Each page represents one observed screen state in the target application.

```jsonc
{
  "id": "page-1",                          // unique page id within this file
  "label": "MouseTester main page",        // short human-readable label (from state_guess)
  "screen_summary": "A minimal mouse-testing utility...",
                                            // full observe summary describing the interface
  "state_guess": "MouseTester main page",   // concise state classification hint
  "screenshot_path": "artifacts/captures/...png",
                                            // optional path to screenshot file
  "timestamp": "15:30:05",                 // observation time (HH:MM:SS)

  "controls": [                            // discovered interactive controls
    {
      "label": "点击此处测试",              // control display text / accessible name
      "type": "button",                     // control kind: button | input | link | toggle | menu | titlebar-button | label | canvas | image
      "description": "Main test trigger button",
                                            // purpose / action description from model
      "bbox": {                             // bounding box (pixels, unscaled)
        "x": 320,
        "y": 240,
        "width": 160,
        "height": 48
      },
      "click_point": {                      // recommended click point (center of bbox by default)
        "x": 400,
        "y": 264
      },
      "status": "clicked",                  // "clicked" | "unclicked"

      // ---- only when status = "clicked" ----
      "click_goal": "点击此处测试",         // the goal used for this click
      "click_screenshot": "artifacts/captures/...png",
                                            // screenshot taken at click time
      "navigated_to_page_id": "page-2",     // id of the page reached after this click

      // ---- only when status = "unclicked" ----
      "possible_navigation": "Opens the test dialog"
                                            // guessed navigation target (optional)
    }
  ]
}
```

### Control types

| type | Description |
|------|-------------|
| `button` | Standard clickable button |
| `input` | Text input / textarea |
| `link` | Hyperlink / anchor |
| `toggle` | Checkbox / radio / switch |
| `menu` | Dropdown / context menu trigger |
| `titlebar-button` | Minimize / maximize / close |
| `label` | Non-interactive text display |
| `canvas` | Drawing / chart area |
| `image` | Static image / icon |

## `transitions[]` — Click transitions

Each transition records a successful click that navigated from one page to another.

```jsonc
{
  "from_page": "page-1",                   // source page id
  "to_page": "page-2",                     // target page id
  "goal": "点击此处测试",                  // the click goal
  "action": "click"                        // action type: "click" | "confirmed_click"
}
```

## Complete example

See `MouseTesterWeb_path_graph.json` for a full working example generated by the browser panel.

## Notes

- Coordinates are in unscaled screenshot pixels; multiply by the runtime scale factor for physical screen coordinates.
- Controls with `status: "unclicked"` are candidates discovered during observe/locate but never executed.
- The `transitions` array maintains chronological navigation order.
- Files are saved both locally (browser localStorage for panel history) and on disk (this folder for sharing / version control).
- To edit a path graph, modify the JSON in this folder, then reload it from the panel history dropdown.
